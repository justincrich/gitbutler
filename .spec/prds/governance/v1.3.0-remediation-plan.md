# Governance PRD v1.2.0 → v1.3.0 — Remediation Plan

This plan folds **13 new findings from a parallel red-hat review** (3 RED `R-NEW-*`, 10 YELLOW `Y-NEW-*`) into the **existing** Phase A/B/C remediation plan, **preserving the wave structure** (A1 semantics · A2 criteria · A3 count-reconciliation · B MEDIUM · C deferred). All work is **PRD documentation only** — no code, consistent with the v1.2.0 precedent ("all on the PRD documentation, no code").

> **Counting principle (load-bearing).** This plan recomputes all headline counts honestly rather than "holding them stable" while editing the AC list. A spec whose AC list grows but whose README still reads the old total is internally inconsistent — which is exactly the failure the A3 count-reconciliation wave exists to prevent. Additions that *make explicit* a previously-implicit behavior are flagged as such in the version history, but the **canonical counts are recomputed to match the actual AC list.**

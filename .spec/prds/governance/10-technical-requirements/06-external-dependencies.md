# External Dependencies

**None required (0 new external dependencies).** The initiative is pure Rust inside the existing GitButler workspace, using crates already present.

| Capability                                                              | Satisfied by                                                                    | New dependency? |
| ----------------------------------------------------------------------- | ------------------------------------------------------------------------------- | --------------- |
| Functional permission types + desugar                                   | new `but-authz` crate (plain Rust enums/sets)                                   | No              |
| Config parsing (`.gitbutler/permissions.toml`, `.gitbutler/gates.toml`) | the `toml` / `serde` stack already in the workspace                             | No              |
| Ref-pinned config read (read a committed blob at a target ref)          | `gix` (GitButler's git library) via existing repo/context helpers               | No              |
| Action-boundary composition                                             | the existing `but-api` `_with_perm` idiom (mirrored as `_with_authz`)           | No              |
| Commit / stack-land entry points                                        | `but-action`, `but-workspace`/`gitbutler-stack`                                 | No              |
| CLI nouns                                                               | the `but` verb modules in `crates/but/src/args/` (docs generated by `but-clap`) | No              |
| Local review record                                                     | `but-db` / `but-forge-storage` (existing)                                       | No              |

## Notes

- **No cryptographic dependency in this slice.** Reviews are attributed by principal identity, not signed (Ed25519 review signing is a deferred layer — see `01-scope.md`). If/when signing is added (or the intermediate HMAC integrity check on `local_review_verdicts`, C3), it reuses GitButler's existing commit-signing/secret stack (`but-secret` + the git signing path), still no _new_ external dependency.
- **No network / forge dependency.** The POC is local; the merge gate reads local review state, not a remote forge API (`but-github`/`but-gitlab` are untouched).
- The fence (git hooks / harness hooks) uses the host git already present; it adds no dependency to GitButler.
- The CLI verb surface lives in `crates/but/src/args/` (the `but` crate), not `but-clap` — `but-clap` is a CLI-**docs** generator (it walks the clap command tree and writes a `cli-docs/` dir), not the verb definitions.

# E2E / Testing Criteria — LPR (v1.5.0)

45 criteria covering 40 LPR ACs across 7 UCs (UC-LPR-05 carries a hand-driven full-local-loop `human-gate`, T-LPR-029h; UC-LPR-01 carries an extra drive-layer-integrity proof T-LPR-043 — self-assignment rejected; UC-LPR-02 carries an extra T-LPR-044 — self-posted+self-resolved thread does not suppress another party's remediation signal; UC-LPR-07 carries two extra capstone proofs — T-LPR-041 forged-vs-empty and T-LPR-042 inverse-cannot-land — beyond its 6 one-per-AC rows). Verification is **real `but-db` + real `but-api` + real `gix` git fixtures**, driven through the **real `but review` CLI verbs** (`request` / `assign` / `comment` / `resolve` / `status`, plus `approve` (shipped) and `request-changes` — whose `changes_requested` write LPR implements, since `request_changes_review` is currently a stub, see LPR-003; the `comment` verb's real write is **also** new LPR work, since the shipped `comment_review` (`forge.rs:569`) is likewise a stub that authorizes `CommentsWrite` then returns `task_contract_invalid` writing nothing — LPR's `post_comment` REPLACES, not wraps, that stubbed path) and asserted on the per-project `but.sqlite` row state, the structured JSON denial on stderr + exit code, and the **merge-gate decision** — the same hand-assertion style as the shipped `commit_gate` / `merge_gate` / `governed_loop` tests (`crates/but/tests/`, `crates/but-api/tests/`), which are **hand-assertion tests (not `insta` snapshots)**. No mocks: a real GitButler-governed repo with committed `.gitbutler/{permissions,gates}.toml`, real principals resolved from `BUT_AGENT_HANDLE` (every governed principal — agent and human — resolves through the same env-keyed path; the **agent-vs-human distinction for the tag comes from the opener principal's declared `kind` in committed `.gitbutler/permissions.toml`, read at the target ref**, NOT from handle-resolution), and real rows in the three new additive tables (`local_review_assignments`, `local_review_comments`, `local_review_meta`).

The **load-bearing gate** is the safe-seam proof (T-LPR-040 → T-LPR-042, UC-LPR-07): assignments and comments are orchestration drive-metadata that **never** flip a land decision; only a distinct `local_review_verdicts` approval-at-head does. The **drive-layer integrity** proofs (T-LPR-043 self-assignment rejected, T-LPR-044 self-resolve does not suppress another party's signal) protect the _reconciler narrative_ (R22) — distinct from the gate, which the safe seam already protects. Any criterion touching agent/probabilistic behavior (the reconciler dispatching, the auto "review-requested" workspace-rule hook) **fixtures the trigger and asserts the ENGINE OUTCOME** (a row written, a state derived, a hook fired) — never agent prose.

Type tags: `[integration-test]` · `[api-contract]` · `[build-gate]` · `[human-gate]` · `[e2e-automated]`. (33 integration-test · 6 api-contract · 3 build-gate · 1 human-gate · 2 e2e-automated.)

**Frozen-aware:** these criteria materialize as **Sprint 07 (LPR)** — the human-directed slot; the existing STEER sprint is renumbered 07→08 (per the user directive, executed in the `/kb-sprint-plan` stage) — and reference **no frozen sprint's tests** (Sprints 01a–06b are unchanged). The merge-gate criteria exercise the **governed review-submission path** only; a direct DB write to `local_review_verdicts` stays untestably forgeable (R6/R18) and is explicitly NOT a path under test.

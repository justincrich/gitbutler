# IDENT-007 — `crates/but/tests/but/command/agent.rs` — snapbox CLI snapshots

**Sprint:** [Sprint 08](./SPRINT.md) · **Agent:** `rust-reviewer` · **Estimate:** 180 min · **Type:** FEATURE · **Status:** READY · **Proposed By:** rust-planner (`--no-specialists`)

## Background

IDENT-005/006 deliver the `but agent` verbs and wiring. This task lands the CLI test suite that locks in the snapshot contract for each verb, modeled on the existing `commit_gate.rs` / `merge_gate.rs` patterns. Snapshot tests are the right tier here because the human-readable output is part of the contract (operators and orchestrators parse it).

**Why it matters.** Without snapshot tests, the output format can drift silently between releases — breaking `but-run-sprint`'s parsing of `but agent register`'s stdout. Snapshot tests catch both behavior regressions AND output-format regressions.

**Current state.** `crates/but/tests/but/command/` has `commit_gate.rs`, `merge_gate.rs`, `governed_merge_cli.rs`, `governed_loop.rs`, `group.rs`, `review_guard.rs`. The established pattern is `env.but(["agent", "register", …]).assert().success().stdout_eq(snapbox::str![…])`.

**Desired state.** `crates/but/tests/but/command/agent.rs` exists with snapshot tests for register (happy + unknown-id), whoami, list (empty + populated), unregister (known + unknown idempotent), and `--help`.

## Critical Constraints

- **MUST** use the existing `env.but([...]).assert()` snapbox pattern (`crates/but/tests/but/command/commit_gate.rs:21` is the template).
- **MUST** use `[..]` or `...` wildcards in snapshots for volatile fields (pid, start_time, expires_at, timestamps). Never hardcode a PID in a snapshot.
- **NEVER** use `std::process::Command::new("git")` for fixture setup — use `env.invoke_bash` / `env.invoke_git` (per `crates/but/AGENTS.md` "CLI Tests").
- **STRICTLY** snapshot both `--help` output AND each verb's behavior — `--help` catches clap-drift; behavior snapshots catch handler regressions.
- **MUST** make snapshots stable across runs: use fixed `--pid`, `--start-time`, `--ttl` values in tests; do NOT use `$$` or `std::process::id()`.

## Specification

**Objective:** Add `crates/but/tests/but/command/agent.rs` with snapbox snapshots for each verb + `--help`.

**Success state:** `cargo test -p but --test agent` passes. Snapshots are checked into `crates/but/tests/but/command/snapshots/agent/*.stdout`. Updating with `SNAPSHOTS=overwrite cargo test -p but --test agent` regenerates them.

```rust
#[test]
fn agent_register_known_id_prints_tuple() -> anyhow::Result<()> {
    let env = but_testsupport::environment("ident-register-happy")?;
    env.invoke_bash(r#"
        mkdir -p .gitbutler
        cat >.gitbutler/agents.toml <<'EOF'
[[agent]]
id = "rust-implementer"
permissions = ["contents:write"]
EOF
        git add .gitbutler/agents.toml
        git commit -m "seed agents"
    "#);
    env.but([
        "agent", "register",
        "--pid", "12345",
        "--start-time", "1730000000",
        "--as", "rust-implementer",
        "--ttl", "1h",
    ]).assert()
      .success()
      .stdout_eq(snapbox::str![[r#"
registered: pid=12345 start_time=1730000000 agent_id=rust-implementer expires_at=[..]
"#]]);
    Ok(())
}
```

**Source (help snapshot):**
```rust
#[test]
fn agent_help_lists_verbs() -> anyhow::Result<()> {
    let env = but_testsupport::environment("ident-help")?;
    env.but(["agent", "--help"]).assert().success().stdout_eq(snapbox::str![[r#"
Usage: but agent <COMMAND>

Commands:
  register    Register a PID as a given agent_id
  unregister  Unregister a PID (idempotent)
  list        List live registrations (or --committed for agents.toml)
  whoami      Resolve THIS process's registration
  help        Print this message or the help of the given subcommand(s)

Options:
  -h, --help  Print help
"#]]);
    Ok(())
}
```

**Anti-pattern:** do NOT assert exit codes or stdout via raw `std::process::Output` fields — go through snapbox `.assert().success().stdout_eq(...)`. The snapbox pattern is the project standard (`crates/but/AGENTS.md` "CLI Tests").

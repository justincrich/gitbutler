# MGMT-IPC-004: Regenerate `packages/but-sdk` (perm/group/status/gates) — SDK build-gate before UI wiring (T-MGMT-034)

> **Red-Hat Remediation (cycle 1):** resolves T4. AC-5's "fails closed" is reframed from a positive-presence grep (which cannot falsify the missing-symbol path) to a DESTRUCTIVE ROUND-TRIP: temporarily exclude one governance command symbol at the source, re-run `pnpm build:sdk`, and assert that symbol is ABSENT from `index.d.ts` AND `pnpm -F @gitbutler/desktop check` FAILS when a UI import references it — proving the pipeline is live (not cached/partial-stub). The source exclusion is then reverted and the surface restored.

## What this does

Runs the canonical SDK generation flow (`pnpm build:sdk && pnpm format`) so `packages/but-sdk/src/generated` gains the governance command symbols + their request/response types (perm/group/branch_gates/status) that MGMT-IPC-001 authored and MGMT-IPC-003 registered, and proves the desktop app type-checks against them. This is the **hard build-gate** (T-MGMT-034) before any MGMT-UI-* task imports the new bindings.

## Why

Sprint 06a · PRD 10-ui-infrastructure (governed-front-end wiring), UC-MGMT-06 (T-MGMT-034) · capability CAP-AUTHZ-01. Per the SDK generation flow (RULES.md), `packages/but-sdk` is **generated** from Rust APIs; the typed governance surface must exist before the UI can type-check.

## How to verify

PRIMARY **AC-1** — `pnpm build:sdk && pnpm format && grep -E 'perm_list|perm_grant|perm_revoke|group_create|group_grant|group_add_member|group_remove_member|group_delete|group_list|branch_gates_read|branch_gates_update|governance_status_read' packages/but-sdk/src/generated/index.d.ts`: the generator runs clean and the generated `index.d.ts` contains all 12 governance command symbols. Full checklist in the spec below.

## Scope

- `packages/but-sdk/src/generated/**` (regenerate ONLY via the generator — `pnpm build:sdk`; commit the output).

> **Generated artifact — never hand-edit.** The generated file bears `// Auto-generated by but-ts. Do not edit manually.`; the `but-ts` generator (walking the `#[but_api]` JSON schemas) is the only writer. This task RUNS the generator; it does not author TS by hand.

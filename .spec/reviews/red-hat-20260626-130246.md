## Executive Summary

The IDENT engine is **genuinely, well-engineered work** — no stubs, no `todo!()`, no `#[ignore]`; the deny-default resolver is correct and fail-closed; the 11 gate callsites are all wired to the registry resolver; the migration verb is byte-exact and idempotent; 77+ tests pass across the suites. The unit/contract layer is solid.

**But the registry identity path — the entire deliverable of Sprints 08–11 — does not function end-to-end in the field.** The `but agent register` CLI (crate `but`) writes its registration to **`agents-runtime.toml`**, while the governed gates (crate `but-api`) read **`agent-registry.toml`**. Two different default filenames, two independently-duplicated path resolvers. In production, a correctly-registered process is still a registry-miss at every gate — so governed actions are either denied outright or silently served by the deprecated `BUT_AGENT_HANDLE` env path the sprints existed to retire. **Every one of the 19 gate tests masks this by pinning `BUT_AGENT_REGISTRY_PATH` to one shared absolute file**, so the default-path divergence is never exercised. This is the textbook "tests green, product broken in the field" failure. The `agent-intel` field migration (IDENT-028) confirms it: its directory contains `agents-runtime.toml`, not the `agent-registry.toml` the gate looks for.

Secondary to that: the new threat-model README **claims a security guarantee the code does not implement** (registry file "mode 0600" + "gitignored"; the real field file is `0644`, world-readable, and tracked in git), and the Sprint 11 skill/cross-repo e2e acceptance criteria are reported toward "done" with **zero captured evidence**.

**Disposition: NEEDS-FIXES. Do not mark the IDENT chain complete until C1 (the filename/path divergence) is fixed and proven by a no-override field test.**

## HIGH+ Confidence Findings (multi-agent agreement or orchestrator-verified)

### 🔴 CRITICAL — C1: `but agent register` and the gate use different registry files (registry path inert in the field)
**Confidence: HIGH (orchestrator-verified against source + field).** Not isolated by any single agent — surfaced by connecting the CLI reviewer's filename, the gate reviewers' filename, and the Sprint-11 field file.

- CLI: `crates/but/src/command/agent.rs:22` → `REGISTRY_FILE_NAME = "agents-runtime.toml"`, written via `resolve_registry_path` (`agent.rs:219-252`).
- Gate: `crates/but-api/src/commit/gate.rs:18` → `AGENT_REGISTRY_FILE_NAME = "agent-registry.toml"`, read via `runtime_registry_path` (`gate.rs:230-247`).
- The two crates **also duplicate** path resolution and `repo_hash` (CLI uses `sha2::Sha256::digest`; gate uses `gix::hash::hasher`).
- **Field proof**: `~/Projects/agent-intel/.gitbutler/` contains `agents-runtime.toml` (CLI-written) — the gate looks for `agent-registry.toml` (absent) → empty registry → `Denial::unregistered` unless `BUT_AUTHZ_ALLOW_ENV_HANDLE=1` re-enables the deprecated env path.
- **Why all tests are green**: every gate-enforcement test (`gate_registry_swap.rs`, `agent_registry.rs`, `commit_gate.rs`) sets `BUT_AGENT_REGISTRY_PATH=<tmp>/agent-registry.toml`. The env override is checked first in *both* resolvers, so CLI and gate point at the same absolute file and the filename divergence is invisible. **No test exercises the real default path (no override) for a register→gate round-trip.**
- **Impact**: the registry-based identity path never governs anything in production. IDENT-028's "governed registry-path commit" did not actually exercise the registry (explains Sprint 11's "enforcement only circumstantial").
- **Fix**: share a single constant + a single path-resolution function across both crates (lift into `but-authz`), and add an integration test that registers via the CLI and enforces at a gate **without** `BUT_AGENT_REGISTRY_PATH` set (real default-path round-trip on both macOS-workdir and XDG paths).

### 🟠 HIGH — H1: README threat model claims `0600` + "gitignored"; the real registry file is `0644` and git-tracked
**Confidence: HIGH (3 agents: security-auditor, Sprint 08 G6, Sprint 11 #1 + orchestrator `git check-ignore`).**
- `crates/but-authz/README.md` (IDENT-023) states the runtime registry is created "mode `0600`, owned by the user" and "gitignored." Repo-wide grep for `0o600|set_permissions|PermissionsExt` returns **only README hits** — no code sets any mode. `git check-ignore` confirms the file is **not** ignored; the field file `agent-intel/.gitbutler/agents-runtime.toml` is `-rw-r--r--` and `git ls-files`-tracked.
- On macOS (GitButler's primary desktop platform, no `XDG_RUNTIME_DIR`), both resolvers fall back to the **in-repo `.gitbutler/` workdir path**, so the registry — the documented spoofing trust-root — is world-readable and committable by default.
- This violates IDENT-023's own MUST ("ground every claim in real source") / NEVER ("never overstate the security guarantee").
- **Fix (cheap, stands alone)**: write the registry `0600`, add a `.gitbutler/.gitignore` entry for it (CLI `register` and/or but-init), and describe the macOS workdir-fallback honestly in the README.

### 🟠 HIGH — H2: Sprint 11 e2e ACs reported toward "done" with zero captured evidence
**Confidence: HIGH (Sprint 11 reviewer; structurally corroborated).**
- T-IDENT-036/037/038, IDENT-027 AC-7, and IDENT-028 AC-1(enforcement)/AC-3 are stamped "BRAIN-REPO-OWNED" and require *running* the skills/gates. There is no run log, skill-test output, or CI artifact in either inspectable repo. IDENT-028's governed commit is asserted by a **commit message**, not a gate receipt.

For a done-claim, an AC with no captured evidence is BLOCKED, not PASS — and C1 means the most important one (registry-path enforcement) would actually fail if run.

### 🟠 HIGH — H3: `but-migrate` skill is half-migrated (breaks its own idempotency guarantee)
**Confidence: HIGH (Sprint 11 reviewer #2).**

## MEDIUM Confidence / Severity Findings

- **M1 — IO-error → empty registry → env fallback** (`gate.rs:204-214`). `registry_load_error_is_io` treats *all* IO errors (EACCES, EISDIR, EIO) as "use empty registry"; with `BUT_AUTHZ_ALLOW_ENV_HANDLE=1` set, making the registry unreadable degrades the gate to the env-handle path. Fix: only `NotFound` → empty; every other IO error → `Err` (fail closed). Bundles with H1. *(security-auditor Finding 1.)*
- **M2 — Sub-second PID-reuse window** (`process.rs:45,163`). `start_time` is whole-second on both Linux (`ticks/HZ`) and macOS (`pbi_start_tvsec`). A PID recycled within the same second collides on `(pid, start_time)` and inherits the dead agent's identity; `stale_registration` only fires on *different* start_times. Fix: use sub-second precision (Linux raw ticks; macOS `pbi_start_tvusec`). *(security Finding 4 + Sprint 08 G4 — 2 agents.)*
- **M3 — `rules.rs` omitted from `ENFORCEMENT_PATHS` honesty-grep.** `crates/but-authz/tests/invariant_build_gates.rs` defines `RULES` but leaves it out of the 9-entry `ENFORCEMENT_PATHS` array, so role-preset/label drift inside `list_workspace_rules_scoped_for_caller` wouldn't be caught by the build gate (the callsite + doc tests still cover the resolver call). *(code-reviewer A-1. NOTE: the security-auditor's Finding 6 claim that `forge.rs` is excluded is **factually wrong** — `FORGE_GUARD` is in the array; `rules.rs` is the real omission.)*
- **M4 — IDENT-013 round-trip tests a reimplemented serializer, not the shipped verb.** `agents_toml_migration.rs:11-46` builds `agents.toml` via a test-local TOML parse+reserialize (`write_agents_toml_from_permissions_wire`), not the production `rewrite_principals_to_agents` text rewrite. A comment-stripping regression in the real verb wouldn't be caught *there* (mitigated by IDENT-011/014 byte-exact CLI tests). Test name overstates coverage. *(Sprint 09 + code-reviewer — 2 agents.)*
- **M5 — Task-contract `verify` strings are no-ops / wrong targets (false-green).** IDENT-002/003/004 `verify` filters (`registry_round_trip`, `ttl_boundary`, `resolve_principal_with_registry`, `start_time_monotonic`) match **zero** tests — real names are `IDENT_00N_*` — so they exit 0 running nothing. IDENT-011/012/014 name a non-existent test target (`--test but::command::agent`) and a non-existent API (`current_start_time()`). A pipeline trusting these gets false-green on a security module. *(Sprint 08 G3 + Sprint 09 — 2 agents.)*
- **M6 — but-init steerer teaches the deprecated, now-denied path.** `but-init/assets/git-but-steerer/handle_check_core.py:147-174` still tells humans to run `but perm list` and `export BUT_AGENT_HANDLE=<principal>` — which the post-Sprint-09/10 engine ignores unless `BUT_AUTHZ_ALLOW_ENV_HANDLE=1`, so a human following it is denied. Stale human-facing surface in the sprint whose purpose is the human-facing identity model. *(Sprint 11 #3.)*
- **M7 — `*_as_fleet_owner` admin functions are `pub` with no in-function auth.** 10 governance-mutating functions in `legacy/governance.rs` are fully `pub`; the fleet-owner session check lives only at the Tauri boundary. **Adjudicated down from the auditor's HIGH**: these are the *pre-existing* fleet-owner write path (outside the IDENT-chain scope), and the "attack" is a future-caller hypothetical with no demonstrated untrusted in-process caller. Valid **defense-in-depth** rec: tighten to `pub(crate)` or a capability token. *(security Finding 3, adjudicated.)*

## Executive Summary

Sprint 08 lands the runtime identity substrate — a `(pid, start_time) → agent_id` `Registry`, per-OS `process_start_time` (Linux `/proc` field-22 + `sysconf(_SC_CLK_TCK)` + btime; macOS `proc_pidinfo(PROC_PIDTBSDINFO)`), a `resolve_principal_with_registry` resolver, and the `but agent register/unregister/list/whoami` CLI. **The orchestrator independently verified the work for real**: `cargo test -p but-authz` ✅ (all bins green) and `cargo test -p but --test but -- agent` ✅ (16/16) against a real git sandbox with real procfs/libproc syscalls and real `gix::lock` atomic TOML writes. No `unimplemented!()`/`todo!()`/"not implemented" in production code. The new resolver is (correctly) **not yet wired** into the ~11 gate callsites — that is Sprint 09's explicit scope, not a stub.

Three adversarial reviewers ran in parallel. Their raw verdicts ranged from "NEEDS_FIXES (3 CRITICAL)" to "PASS WITH MINOR CONCERNS." **After orchestrator triage, most "CRITICAL" ratings were severity inflation or factual errors** (resolved with evidence in §Contradictions). The genuine, agreed findings are two test-coverage gaps (`whoami` happy path, bad-`BUT_AGENT_REGISTRY_PATH`) and one unnecessary lint suppression. One legitimate security-design note (registration is not authenticated) survives triage as a documented threat-model item, not a Sprint-08 defect.

**Net:** ship it; address the 3 follow-ups (two small tests + remove one `#![allow]`) before Sprint 10 makes the registry the primary identity source.

## MEDIUM Confidence Findings (2 agents agree)

- [ ] **F-3. No test locks the bad-`BUT_AGENT_REGISTRY_PATH` contract (IDENT-005 AC-6).** | Severity: **Medium (coverage)**
      AC-6 requires a bad registry path to exit 2 naming the path. The handler logic exists (`write_registry_or_exit` → `exit_registry_write_error` → `std::process::exit(2)`, `command/agent.rs:198-217`), but no snapshot test enforces it. Without a test the exit code / message can silently regress.
      Agents: rust-reviewer, security-reviewer, orchestrator.
      Fix: add a snapshot test pointing `BUT_AGENT_REGISTRY_PATH` at a path whose parent does not exist and asserting exit 2 + path-in-stderr.

- [ ] **F-4. Registration is not authenticated — any process that can run `but` in the repo can register as any agent_id in committed `agents.toml`.** | Severity: **Medium (threat-model) — documented limitation, not a defect.**
      `but agent register --as <id>` validates the id exists in `.gitbutler/agents.toml` (`command/agent.rs:38-44`) but does not verify the caller's right to claim it. The `--by` field is operator-supplied and unverified. In a single-user dev workflow this is acceptable (the operator *is* the trust root); in multi-user or CI-shared checkouts it is an escalation primitive.
      Agents: security-reviewer (raised); orchestrator (severity-triaged).
      Fix: document explicitly in the IDENT PRD/`agents.toml` spec that Sprint 08 registration is operator-trusted and that multi-tenant enforcement is out of scope until a later sprint; consider audit-logging registration events (pairs with the Sprint 07 telemetry carrier).

- [ ] **F-5. PID-reuse defense granularity is ~1 second on both macOS and Linux (documented).** | Severity: **Low-Medium (threat-model).**
      The registry key is `(pid, u64 unix_seconds)`. macOS `process.rs:163` returns `pbi_start_tvsec` only (discards `pbi_start_tvusec`); Linux computes `boot_time + start_ticks/CLK_TCK` with integer division → seconds. The module doc (`process.rs:3-18`) states this explicitly: *"the canonical form is Unix seconds."* Two processes reusing a PID *within the same wall-clock second* after boot rollover would collide. Window is small; design is intentional and consistent across platforms.
      Agents: security-reviewer (raised); orchestrator (confirmed documented intent).
      Fix: record in the threat model; revisit if sub-second churn becomes plausible. No code change needed for Sprint 08.

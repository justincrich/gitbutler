## Findings that survive on `origin/master` (the real remaining work)

### 🔴 BLOCKER-1 (MEDIUM/HIGH) — GATES AC-5: commit gate is at the API entrypoint, not the engine narrow waist
- **Group**: GATES · **Status on master: CONFIRMED OPEN** (overturned PASS→PARTIAL by the adversarial pass, re-verified by orchestrator).
- **Spec**: UC-GATES-01 AC-5 + `06-uc-gates.md:21` mandate a *mechanism-agnostic* gate at the commit narrow waist — *"a gate wired to one mechanism would be bypassed by another"* — and **explicitly name the TUI** as a caller that must route through the gate.
- **Reality**: the gate (`enforce_commit_gate`) fires in `crates/but-api/src/commit/create.rs`, an **entrypoint**, not in the waist `but_workspace::commit::commit_create`. On `origin/master` that waist has 7+ direct callers, two of which reach it ungoverned:
  - `crates/but-action/src/simple.rs:118` — the **autonomous agent auto-commit** path. `but-action` does **not depend on `but-authz`/`but-api`** (verified on master). This is the agent-facing mechanism central to the LOOP demo.
  - the **TUI** commit path via `but-transaction` (`crates/but-transaction/src/lib.rs:494`).
- **Why it matters**: neither bypass is carved out as an accepted leak (only R1 fence, R6 review store, R14 N-API, forge-merge are). The structural `assert_gate_helper_call_count` test counts gate-helper occurrences at the entrypoints and **cannot detect an ungated engine caller** — it's test-theatre for the anti-bypass guarantee specifically.
- **Fix**: gate at the engine waist (`commit_create`) *or* wire `enforce_commit_gate` into the `but-action` and TUI paths, **and** add a build-gate grep asserting every `commit_create` caller is gate-preceded (mirror the N-API/R14 audit). *Caveat: the common path commits to feature branches; the protected-branch bypass is reachable but not the default flow — hence MEDIUM/HIGH, not CRITICAL.*

## Accepted POC limitations — confirmed present-and-honest (NOT counted as gaps)

These are deliberate scope edges, documented in `01-scope.md` / `00-overview.md`, not silently shipped as boundaries:

1. **Git/hook fence (R1)** — governance binds the governed `but` path; raw `git` / `--no-verify` / `core.hooksPath` are outside. *Irrigation thesis: cheaper-than-defection, not impossible.*
2. **R6 forgeable local review store (High)** — `local_review_verdicts` is not tamper-proofed against a direct DB write; tests drive only the governed path. Disclosed.
3. **R14 N-API runtime residual** — distinct from the **build-gate test** deliverable, which now exists (`lpr_010_audit.rs`).
4. **Forge-locality of merge** — a forge-bound merge can't land on a credential-less local fixture (drives ADVISORY-2).
5. **Single trusted human / fleet-owner** — personal-tenant trust; agents stay functionally bound.
6. **`config.invalid` fail-closed** and **inert-until-committed** ref-pin semantics — security properties, not bugs.
